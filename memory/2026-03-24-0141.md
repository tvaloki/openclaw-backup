And if you want the full clean setup for separate `main`, `builder`, and `maintainer` Telegram accounts/agents, the doc-aligned flow is:

```bash
openclaw agents add builder --workspace ~/.openclaw/workspace-builder --non-interactive
openclaw agents add maintainer --workspace ~/.openclaw/workspace-maintainer --non-interactive

openclaw channels add --channel telegram --account builder --token '<BUILDER_TELEGRAM_BOT_TOKEN>'
openclaw channels add --channel telegram --account maintainer --token '<MAINTAINER_TELEGRAM_BOT_TOKEN>'

openclaw agents bind --agent main --bind telegram:default
openclaw agents bind --agent builder --bind telegram:builder
openclaw agents bind --agent maintainer --bind telegram:maintainer

openclaw gateway restart
```

A couple of important doc-backed notes:
- Adding a non-default Telegram account migrates the existing single-account config into `channels.telegram.accounts.default`.
- Non-interactive `channels add` does **not** auto-bind; that’s why the explicit `agents bind` commands matter.
