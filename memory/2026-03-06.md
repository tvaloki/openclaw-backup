## Supabase PSQL Connection Troubleshooting (Mac)
- **Context:** Attempted to run a bulk CSV import (`\copy`) to Supabase `public.thoughts` using `psql` from a local Mac.
- **Connection String Issues:** Standard URIs (`postgresql://...`) failed with `server closed the connection unexpectedly`. Switched to key/value connection strings (`host=... port=... user=... dbname=... sslmode=require`) to bypass URI parsing bugs.
- **IPv6 Routing Failure:** Direct DB connection (`db.fkprogqogwguxdcvdciw.supabase.co`) failed with "No route to host". Diagnosed as IPv6 routing failure because Mac environment couldn't resolve IPv4 A records via `dig`.
- **Pooler Auth Format Mismatch:** Pooler connections (port 6543) failed with `FATAL: password authentication failed for user "postgres"`. Identified that the pooler requires the `postgres.<project-ref>` username format, whereas direct connections (port 5432) use just `postgres`.

## Bulletproof Outlook Notifications on Mac (Service Central)
- **Problem:** Need 100% reliable, immediate notifications for Service Central emails using Outlook 365 on a Mac.
- **Why Mac rules fail:** Local macOS Outlook rules are unreliable for critical alerts. "App Nap," sleep mode, and the "New Outlook for Mac" (which killed AppleScript support) will cause you to miss emails. Local client rules fail if the client isn't active.
- **The Solution (Power Automate):** Move the trigger off the Mac and onto the Exchange Server using Microsoft Power Automate. It runs 24/7 regardless of the Mac's power state.
- **Setup Steps:**
  1. Go to make.powerautomate.com and log in with Parallon/O365 credentials.
  2. Create an "Automated cloud flow".
  3. Trigger: "When a new email arrives (V3) - Office 365 Outlook".
  4. Configure Trigger: Folder = Inbox, From = [Service Central email].
  5. Action: "Send me a mobile notification" (requires the Power Automate app on the phone).
  6. Text: "URGENT: Service Central Ticket - @{triggerOutputs()?['body/subject']}".
- **Advantages:** Zero client dependency (Mac can be asleep/off), bypasses Mac Focus Modes via mobile push notification, and can be escalated to SMS via carrier gateways (e.g., number@vtext.com) if needed.
