## Supabase PSQL Connection Troubleshooting (Mac)
- **Context:** Attempted to run a bulk CSV import (`\copy`) to Supabase `public.thoughts` using `psql` from a local Mac.
- **Connection String Issues:** Standard URIs (`postgresql://...`) failed with `server closed the connection unexpectedly`. Switched to key/value connection strings (`host=... port=... user=... dbname=... sslmode=require`) to bypass URI parsing bugs.
- **IPv6 Routing Failure:** Direct DB connection (`db.fkprogqogwguxdcvdciw.supabase.co`) failed with "No route to host". Diagnosed as IPv6 routing failure because Mac environment couldn't resolve IPv4 A records via `dig`.
- **Pooler Auth Format Mismatch:** Pooler connections (port 6543) failed with `FATAL: password authentication failed for user "postgres"`. Identified that the pooler requires the `postgres.<project-ref>` username format, whereas direct connections (port 5432) use just `postgres`.
- **Action Items:** User was instructed to rotate their Supabase DB password (exposed during debugging) and to use the exact connection string from the Supabase dashboard (Project Settings -> Database) matching the correct user format for the chosen port.
